# 信息技术学院 - 学员评分管理系统

一个面向手机端用户的 **无后端** 学员评分管理系统，采用 **Atom One Dark** 暗色主题，所有数据存储在浏览器本地。系统集成了学员评分、学员管理、数据总览三大核心模块，支持一键复制信息、客观评分、自动拼音生成、双重确认删除等实用功能。

## 功能特点

### 📋 学员评分页面
- **卡片化展示**：所有在读学员以卡片形式呈现，清晰显示学号、姓名、拼音、班级、责任区及当日状态胶囊。
- **状态胶囊**：签到、动员笔记、结业小结状态一目了然，绿色表示已达成，红色表示未达成。
- **详情评分页**：点击卡片进入详情，可切换日期查看/编辑历史评分。
- **智能签到**：出勤状态选择“正常”或“迟到”时自动标记为已签到；选择“病假”或“旷班”自动标记为未签到。
- **客观评分**：大部分卫生评分项（黑版槽、桌椅、抽屉清洁、地面清洁）采用“达标/未达标”快捷切换。
- **额外加分与上限**：支持 0~5 分额外加分并填写说明；总分上限 100 分，超出部分自动截断。
- **一键复制**：可复制学号姓名组合（如 `2424001张三`），任意信息文本长按/点击即可复制。

### 👥 学员管理页面
- **智能导入**：填写姓名自动生成带声调的拼音（不可编辑）；提交后仅清空姓名和手机号，方便连续添加。
- **严格防重复**：学号、手机号、责任区任一重复都会提示异常，阻止导入。
- **学员编辑**：可修改学员所有基本信息，拼音随姓名自动更新。
- **重修机制**：标记为“重修”的学员将从评分页面隐藏，但仍保留历史数据。
- **安全删除**：支持单个学员删除或清空全体学员，操作均需**双重确认**，防止误删。

### 📊 数据总览页面
- **按人查看**：通过顶部可横向滚动的姓名标签切换学员。
- **历史记录**：按日期倒序展示每日得分，并用彩色标签区别病假、旷班、迟到、正常状态。
- **平均分与排名**：自动计算该学员的历史平均分（保留两位小数）及在所有在读学员中的排名。
- **快捷复制**：分数和姓名均支持点击复制。

## 使用说明

1. 直接用手机或电脑浏览器打开 `index.html` 文件。
2. 底部导航栏切换三个页面：学员评分、学员管理、数据总览。
3. 首次使用会自动生成演示数据（可手动清除）。
4. 所有操作自动保存到浏览器本地存储（localStorage），关闭页面或刷新后数据不丢失。

### 学员导入流程
- 进入「学员管理」页，填写姓名、学号、班级、手机号、责任区。
- 姓名的拼音会自动生成并显示在拼音输入框（只读）。
- 点击「导入学员」即完成添加，若学号/手机号/责任区中有任意一项与现有学员重复，会弹出错误提示。

### 日常评分流程
- 在「学员评分」页点击任一学员卡片，进入详情。
- 可通过顶部按钮切换日期。
- 选择出勤状态（正常/病假/迟到/旷班）自动关联签到。
- 分别对各卫生项目选择达标/未达标，黑板等第下拉选择，如有额外加分填写分数和说明。
- 点击「保存评分」即可。

### 查看总览
- 进入「数据总览」页，点击上方学员姓名切换视图。
- 列表展示该学员每日最终得分及特殊标记。
- 底部显示平均分和当前排名（仅统计在读学员）。

## 技术栈

- 纯前端实现，无需任何后端或数据库。
- **HTML5 + CSS3 + Vanilla JavaScript**
- 使用 `pinyin-pro` 库实现中文转拼音（含声调）。
- 数据持久化：`localStorage`
- 响应式布局，适配手机屏幕，支持深色主题。

## 注意事项

- 数据仅保存在当前浏览器的本地存储中，**更换浏览器或清除浏览器数据将导致数据丢失**。建议定期手动备份数据（可自行添加导出功能）。
- 删除学员或清空数据为不可逆操作，请谨慎执行。
- 签到状态每天自动重置（根据出勤状态），笔记和小结状态自上交日起永久有效。
- 总分超过 100 分时自动按 100 分计算。
